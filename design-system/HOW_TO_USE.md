# Как сгенерировать экран в дизайне mymeet.ai через Claude

Эта инструкция для дизайнеров и продактов команды. Цель — научиться за 10 минут получать полноценные интерактивные прототипы экранов в стиле mymeet.ai через диалог с Claude.

---

## Что ты получишь в результате

1. **Интерактивный прототип** на `localhost:3000/<your-screen>` — можно потыкать, проверить интеракции, показать стейкхолдерам.
2. **Деплой на GitHub Pages** — публичная ссылка, которую можно скинуть в чат: `https://avr1l14th.github.io/design-lab/<your-screen>`.
3. **Опционально:** экспорт прототипа обратно в Figma как фрейм (для финальной доработки руками).

Всё это — в стиле mymeet.ai: правильные цвета, шрифты, отступы, компоненты.

---

## Разовая настройка (~10 минут)

### 1. Установи Claude Code

```bash
curl -fsSL https://claude.ai/install.sh | bash
```

После установки запусти `claude` в терминале — он попросит залогиниться через браузер.

### 2. Подключи Figma MCP

Установи [Figma desktop app](https://www.figma.com/downloads/), залогинься в свой аккаунт и убедись, что у тебя есть доступ к команде **mymeet.ai** в Figma.

В терминале запусти:
```bash
claude mcp add figma
```
Следуй инструкциям. После настройки Claude сможет читать макеты mymeet.ai через Figma desktop.

### 3. Склонируй design-lab

```bash
cd ~
git clone https://github.com/avr1l14th/design-lab.git
cd design-lab
npm install
```

Если у тебя нет push-доступа — попроси Фёдора (дизайнера) добавить тебя в коллабораторы репо.

### 4. Запусти Claude в этой папке

```bash
cd ~/design-lab
claude
```

Готово. Теперь можно генерировать.

## Как сгенерировать экран

### Самый быстрый способ — через skill

В этом репо есть готовый skill `design-screen`. Просто скажи Claude:

```
/design-screen сделай экран настроек профиля
```

Или просто:
```
Сделай в нашем дизайне экран настроек профиля
```

Claude автоматически активирует skill (по фразе «в нашем дизайне / mymeet / прототип») и выполнит **полный цикл**:
1. Прочитает `DESIGN_SYSTEM.md` и `figma-library-snapshot.json` (реестр всех компонентов либы)
2. Создаст `app/<slug>/page.tsx` с использованием React-компонентов, чьи имена 1-в-1 как в Figma-либе
3. Возьмёт только разрешённые значения spacing/radius/font-size (не «20» если в либе есть только «16, 24»)
4. Зарегистрирует прототип в `prototypes-registry.ts`
5. Запустит dev server, проверит экран
6. (По запросу) экспортирует в Figma — НЕ плоский ватман, а **фрейм с инстансами компонентов либы** и привязанными токенами (text styles, spacing/radius/color variables)

### Шаблон промпта (минимальный)

```
Сделай в нашем дизайне экран <тут описание>.
```

Если нужен только локалхост (без экспорта в Figma):
```
Сделай в нашем дизайне экран X. В Figma пока не экспортируй.
```

Если хочешь сразу с экспортом:
```
Сделай экран X в нашем дизайне и закинь в Figma.
```

### Примеры рабочих промптов

**Простой:**
> Сделай в нашем дизайне экран настроек профиля пользователя — аватар, имя, email, переключатели уведомлений, кнопка «Сохранить».

**С отсылкой к существующему:**
> Сделай в нашем дизайне экран команды (workspace members) — таблица с колонками Имя, Email, Роль, Статус. Структура и стиль как в `app/search-filters`. Добавь поиск по имени и фильтр по роли.

**С полным циклом до Figma:**
> Сделай экран биллинга (план + история платежей + кнопка апгрейда) и сразу экспортируй в Figma инстансами библиотеки.

**С привязкой к Figma:**
> Сделай в нашем дизайне экран онбординга — 4 шага. Я открыл в Figma desktop фрейм 1234:5678 — прочитай его и используй как референс для структуры.

### Что Claude сделает автоматически (по skill `design-screen`)

1. **Контекст:** прочитает `DESIGN_SYSTEM.md` + `figma-library-snapshot.json` (89 компонентов, 14 text styles, ~120 variables)
2. **Localhost:** создаст `app/<slug>/page.tsx` с компонентами, чьи имена и варианты 1-в-1 как в Figma (`SidebarItem`, `MeetingListItem`, `TabSegmented`, `Logo`, `Integrations/Telemost/16` и т.д.)
3. **Только токенные значения:** spacing из {0,2,4,6,8,12,16,24,32,40,64,80}, radius {0,1,2,3,4,full}, font sizes из либы
4. **Регистрация:** обновит `prototypes-registry.ts`
5. **Проверка:** запустит dev, проверит на скриншоте, очистит ошибки
6. **(Опционально) Figma export:**
   - Захватит layout-reference через `generate_figma_design`
   - Соберёт параллельный фрейм через `use_figma` с инстансами либы
   - Привяжет text styles + spacing/radius/color variables (≥300 биндингов на медиум-экран)
   - Audit: snap любого non-token значения к шкале
7. **Отчёт:** localhost URL, Figma URL, список инстансов, число биндингов, скриншот

## Обновление либы

Если ты добавил/изменил компоненты в Figma — скажи Claude:
```
обнови snapshot
```
или
```
обновил либу, переснифай
```

Skill `refresh-figma-snapshot` перепишет `figma-library-snapshot.json` свежими данными из Figma. После этого новые компоненты сразу подхватываются в `/design-screen`.

### Как проверить и докрутить

1. Открой `http://localhost:3000/<your-slug>` в браузере.
2. Если что-то не так — попроси Claude поправить:
   > Кнопка «Сохранить» должна быть полной ширины. Заголовок 24px, не 32px. Источники в две колонки.
3. После каждой правки Claude перезапустит проверку. HMR работает — ничего вручную перезапускать не нужно.

### Деплой

После того как экран понравился — сделай `git commit` и `git push` через Claude:
> Закоммить изменения и запушь в main.

GitHub Action автоматически задеплоит на Pages за 1–2 минуты. Ссылка появится тут: `https://avr1l14th.github.io/design-lab/<your-slug>`.

## Экспорт в Figma

Когда экран готов и нравится — можно вытащить его обратно в Figma как фрейм:

1. В Claude скажи:
   > Экспортируй текущий экран в Figma через `generate_figma_design`. Положи в наш файл mymeet.ai.

2. Claude захватит рендер с `localhost:3000/<your-slug>` и создаст новый фрейм в файле mymeet.ai (или новый файл — он спросит).

3. Открой Figma — фрейм появится. **Важно:** это плоский векторный слой, не подключённые компоненты. Чтобы превратить в инстансы вашей библиотеки, придётся переподключить кнопки/инпуты руками. Зато **геометрия, цвета и типографика — пиксель-в-пиксель.**

## Когда что-то идёт не так

### «Claude использует не те цвета»

Скорее всего, он пропустил `DESIGN_SYSTEM.md`. Скажи прямо:
> Прочитай `design-system/DESIGN_SYSTEM.md` и используй только цвета из объекта `tokens`. Никаких произвольных hex-значений.

### «Стиль не похож на mymeet.ai»

Покажи ему эталон:
> Сверстай по аналогии с `app/search-filters/page.tsx` — те же отступы, те же размеры шрифтов, тот же sidebar.

### «Хочу точно как в Figma, а получилось по-другому»

Открой Figma desktop, выдели нужный фрейм, и попроси Claude его прочитать:
> Я выделил в Figma desktop фрейм с этим экраном. Прочитай его через `mcp__figma__get_design_context` и адаптируй вёрстку под него.

### «Сборка падает»

Попроси Claude посмотреть логи:
> Что в логах сервера? Почему билд падает?

Не пытайся чинить сам — Claude справится.

### «Не работает Figma MCP»

Открой Figma desktop. Закрой и открой заново. Убедись, что ты залогинен в нужный аккаунт (Fedor).

## Правила хорошего тона

1. **Один экран — одна папка `app/<slug>/`.** Не трогай чужие папки.
2. **Сначала поиграйся локально**, потом коммить. Если экран сырой — оставь его в локальной ветке, не пушь в main.
3. **Не редактируй `DESIGN_SYSTEM.md` сам.** Если нашёл ошибку — напиши Фёдору, он обновит для всех.
4. **Не пуши прямой коммит на main с экспериментами.** Используй ветку `feature/<your-slug>` и Pull Request, если есть сомнения.
5. **Если сделал что-то крутое — расскажи команде.** Это инвестиция в общий навык.
