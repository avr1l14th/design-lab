# Skill «mymeet-design» — генерация чего угодно в дизайне mymeet.ai

> Документ для шеринга любому члену команды. Self-contained skill — работает в любой папке, любом проекте, не требует клонирования каких-либо репо.

---

## Что это

`mymeet-design` — это **Claude Code skill** который применяет правила, токены и философию дизайн-системы mymeet.ai к **любому** артефакту, который ты попросишь сделать.

Когда ты пишешь Claude «сделай в нашем дизайне X» или «обнови X под mymeet» — он:

1. Применяет mymeet-философию (типографика Inter Medium 13 как UI body, brand blue только в accent'ах, border-only cards без shadow, filled heroicons, плотность Stripe-style, ToV без «Вы» с большой)
2. Использует токены DS (цвета, spacing scale 0–80, radius 0–4, font sizes 10–32)
3. Подключает реестр Figma-компонентов библиотеки (~89 компонентов)
4. Выбирает технологию под задачу и генерит результат

**Артефакт может быть любым** — UI экран, документ, презентация, рассылка, лендинг, диаграмма, social post, slide-карточка, баннер, иллюстрация, чек-лист, what-have-you. Skill не ограничен списком форматов.

Главное — результат выглядит **родным** для всех остальных артефактов команды.

## Установка (один раз, ~5 минут)

### 1. Claude Code

Если ещё нет — поставь Claude Code:
```bash
curl -fsSL https://claude.ai/install.sh | bash
```
Запусти `claude`, залогинься через браузер.

### 2. Figma MCP (опционально, но рекомендую)

Нужно если будешь экспортировать в Figma или хочешь чтобы skill читал live-данные либы.

Поставь [Figma desktop app](https://www.figma.com/downloads/), залогинься, убедись что есть доступ к команде mymeet.ai.

Подключи MCP:
```bash
claude mcp add figma
```

### 3. Сам skill

Один из двух способов:

**Способ A — через git (рекомендую):**
```bash
mkdir -p ~/.claude/skills
cd ~/.claude/skills
git clone --filter=blob:none --sparse https://github.com/avr1l14th/design-lab.git tmp-design-lab
cd tmp-design-lab
git sparse-checkout set .claude/skills/mymeet-design
mv .claude/skills/mymeet-design ~/.claude/skills/
cd .. && rm -rf tmp-design-lab
```

Это копирует **только** папку skill (без всего design-lab репо). Skill окажется в `~/.claude/skills/mymeet-design/`.

**Способ B — скачать ZIP:**
1. Открой https://github.com/avr1l14th/design-lab/tree/main/.claude/skills/mymeet-design
2. Скачай файлы (`SKILL.md` и `figma-library-snapshot.json`) кнопкой Download или через UI
3. Положи их в папку `~/.claude/skills/mymeet-design/` (создай если её нет)

### 4. Проверь

Запусти Claude Code где угодно (не обязательно в design-lab):
```bash
cd ~/Desktop          # или любая другая папка
claude
```

Введи `/skills` или начни печатать `/mymeet-` — должно появиться `/mymeet-design`.

## Как пользоваться

### Через слеш-команду (надёжный путь)

```
/mymeet-design <твой запрос>
```

Примеры:
- `/mymeet-design сделай экран настроек профиля`
- `/mymeet-design обнови этот документ под наш дизайн` (приложи PDF/DOCX к чату)
- `/mymeet-design собери презентацию про X в нашем стиле`
- `/mymeet-design свёрстай страницу про фичу транскриптов`
- `/mymeet-design сделай шаблон письма пользователям`
- `/mymeet-design нарисуй диаграмму процесса в нашем стиле`
- `/mymeet-design свёрстай чек-лист в стиле mymeet`

### Без слеша (auto-invoke по фразе)

Skill активируется автоматически если в запросе есть «в нашем дизайне», «mymeet», «в стиле mymeet», «переверстай под наш дизайн»:

```
Сделай в нашем дизайне экран статистики встреч
Обнови этот отчёт под mymeet
Собери slide-карточку про новую интеграцию в нашем стиле
```

Если auto-invoke не сработал — вызови явно через `/mymeet-design`.

## Что Claude сделает

1. **Контекст** — прочитает SKILL.md (все правила inline в нём) и `figma-library-snapshot.json` рядом (если артефакт использует компоненты либы).
2. **Решение** — определит какая технология лучше подходит под твой запрос (Next.js, HTML с print-стилями, Figma фрейм с инстансами либы, table-based email, SVG, что угодно ещё) — выбор делает он сам по смыслу задачи.
3. **Спросит куда сохранить** если неясно — текущая папка / Desktop / Figma / clipboard / etc. Не предполагает структуру проекта.
4. **Генерация** — собирает артефакт с правильной типографикой (Inter), цветами из токенов, spacing из шкалы, иконками filled 16/20, mymeet ToV.
5. **Output** — путь до файла / Figma URL / preview, скриншот, summary что использовано из либы.

## Где работать

Skill **self-contained** — не привязан к какому-либо репо или проекту. Запускай Claude в любой папке:

- На Desktop — чтобы сделать одноразовый артефакт
- В своём рабочем проекте — чтобы добавить что-то в стиле mymeet
- В новой папке для конкретного клиента/задачи

Skill сам разберётся куда сохранять файлы (спросит или предложит разумный default).

## Если что-то не так

| Проблема | Что делать |
|---|---|
| `/mymeet-design` не в списке `/skills` | Проверь что папка `~/.claude/skills/mymeet-design/` существует и в ней есть `SKILL.md`. Перезапусти Claude Code (`/exit` + `claude`). |
| Skill не активируется auto — Claude генерит «по своему» | Вызови явно через `/mymeet-design` |
| Нет нужного компонента в либе (Claude сам скажет) | Передай задачу дизайнеру (Фёдору) добавить компонент в Figma либу, либо разреши Claude сделать custom-вариант |
| Snapshot устарел (новые компоненты не подхватываются) | Скажи `обнови snapshot` — Claude перетянет свежие данные из Figma через MCP и перепишет `figma-library-snapshot.json` в твоей skill папке. Или скачай файл руками (Способ B из установки). |
| Что-то не в стиле mymeet (stroke иконки, transition анимации, неправильный sidebar) | Попроси Claude поправить с указанием конкретного правила. Skill знает все hard rules. |
| Хочешь обновить skill полностью | Повтори шаг 3 установки — это перезапишет файлы свежими. |

## Что внутри skill (если интересно)

Открой `~/.claude/skills/mymeet-design/SKILL.md` чтобы посмотреть полную философию, hard rules, цвета, типографику, common pitfalls. Это документ который Claude читает каждый раз когда skill активируется.

В `~/.claude/skills/mymeet-design/figma-library-snapshot.json` — полный реестр Figma-компонентов с их keys, variants и текстовыми слоями. Открой если хочешь увидеть какие готовые компоненты доступны.

## Принцип короче всего

> Skill не привязан к конкретному формату output или проекту. Он применяет mymeet-принципы (токены, типографика, плотность, цветовая дисциплина, ToV, референсы Linear/Notion/Vercel) к **любому** артефакту, который ты попросишь сделать. Технологию для output Claude выбирает сам исходя из задачи. Работает в любой папке.

То есть ты говоришь **что** нужно — Claude разбирается **как** это лучше всего сделать в нашем дизайне.
