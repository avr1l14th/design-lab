# Skill «mymeet-design» — генерация чего угодно в дизайне mymeet.ai

> Документ для шеринга любому члену команды.

---

## Что это

`mymeet-design` — один универсальный **Claude Code skill** для проекта design-lab.

Когда ты пишешь Claude «сделай в нашем дизайне X» или «обнови X под mymeet» — он автоматически:

1. Читает правила и токены дизайн-системы mymeet.ai
2. Подключает реестр Figma-компонентов библиотеки
3. Применяет mymeet-философию (типографика, цвета, плотность, ToV, референсы Linear/Notion/Vercel/etc)
4. Выбирает технологию под задачу и генерит результат

**Артефакт может быть любым** — UI экран, документ, презентация, рассылка, лендинг, отчёт, диаграмма, social post, slide, чек-лист, what-have-you. Skill не ограничен списком форматов — он умеет применять одни и те же принципы к чему угодно.

Главное — результат выглядит **родным** для всех остальных артефактов команды, потому что использует общие токены, типографику и философию.

## Что нужно поставить (один раз, ~15 минут)

### 1. Claude Code (CLI от Anthropic)
```bash
curl -fsSL https://claude.ai/install.sh | bash
```
Запусти `claude`, залогинься через браузер.

### 2. Figma Desktop App
Скачай: https://www.figma.com/downloads/. Залогинься в свой аккаунт. Убедись что есть доступ к команде **mymeet.ai** в Figma.

### 3. Подключи Figma MCP к Claude
```bash
claude mcp add figma
```

### 4. Склонируй design-lab
```bash
cd ~/Claude
git clone https://github.com/avr1l14th/design-lab.git
cd design-lab
npm install
```

Если нет push-доступа — попроси добавить тебя в коллабораторы.

### 5. Запусти Claude в этой папке
```bash
cd ~/Claude/design-lab
claude
```

Проверь что skill подгружен — введи `/skills` или начни печатать `/mymeet-` — должна появиться подсказка `/mymeet-design`.

## Как пользоваться

### Через слеш-команду (надёжный путь)

```
/mymeet-design <твой запрос>
```

Примеры:
- `/mymeet-design сделай экран настроек профиля`
- `/mymeet-design обнови этот документ под наш дизайн` (приложи PDF/DOCX к чату)
- `/mymeet-design собери презентацию про X в нашем стиле`
- `/mymeet-design свёрстай страницу про фичу транскриптов`
- `/mymeet-design сделай шаблон письма пользователям`
- `/mymeet-design нарисуй диаграмму процесса в нашем стиле`
- `/mymeet-design свёрстай чек-лист в стиле mymeet`

### Без слеша (auto-invoke по фразе)

Skill активируется автоматически если в запросе есть фразы вроде «в нашем дизайне», «mymeet», «в стиле mymeet», «переверстай под наш дизайн». Просто пиши естественно:

```
Сделай в нашем дизайне экран статистики встреч
Обнови этот отчёт под mymeet
Собери slide-карточку про новую интеграцию в нашем стиле
```

Если по какой-то причине auto-invoke не сработал и Claude генерит «по своему» — пиши явно через `/mymeet-design`.

## Что Claude сделает

1. **Контекст** — прочитает правила дизайн-системы (`design-system/DESIGN_SYSTEM.md`) и реестр Figma-компонентов (`design-system/figma-library-snapshot.json`).
2. **Решение** — определит какая технология лучше подходит под твой запрос (Next.js страница в репо, HTML с print-стилями для документа, Figma фрейм с инстансами либы, table-based email, SVG для диаграммы, etc.) — выбор делает он сам по смыслу задачи, не по жёсткому списку.
3. **Генерация** — собирает артефакт с правильной типографикой (Inter, Medium 13 как UI-body, Medium 24 для headings, Semibold 32 для hero numbers), цветами из токенов (action/primary blue в CTA и accent'ах, text/primary, surface/page, etc.), spacing из шкалы (0, 2, 4, 6, 8, 12, 16, 24, 32, 40, 64, 80), иконками filled 16/20 (heroicons), и mymeet ToV (деловой живой, без «Вы» с большой).
4. **Output** — путь к локальному preview, финальный артефакт (HTML/Figma URL/файл), скриншот, summary что использовано из либы и что custom.

## Если что-то не так

| Проблема | Что делать |
|---|---|
| Skill не активировался — Claude генерит без skill workflow | Вызови явно через `/mymeet-design` |
| Skill не в списке `/skills` | Перезапусти Claude Code (`/exit` + `claude`) |
| Нет нужного компонента в либе (Claude сам скажет) | Передай задачу дизайнеру (Фёдору) добавить, либо разреши Claude сделать custom-вариант |
| Дизайнер обновил либу — нужно подхватить новые компоненты | Скажи `обнови snapshot` — Claude запустит skill `/refresh-figma-snapshot` |
| Что-то не в стиле mymeet (например stroke иконки вместо filled, или transition на hover, или sidebar 240 вместо 280) | Попроси Claude поправить с указанием конкретного правила. Skill знает все hard rules — обычно достаточно одного указания на конкретную ошибку. |
| Результат не такой как ожидал по сути (не косметика, а логика) | Расскажи Claude что не так и попроси переделать. Skill итеративный — можно переделывать в том же чате. |

## Где живут актуальные правила

- **Skill (что Claude читает)**: `.claude/skills/mymeet-design/SKILL.md` — там полная философия, hard rules, цвета, типографика, common pitfalls. Открой если нужно понять, по каким правилам Claude работает.
- **Дизайн-правила**: `design-system/DESIGN_SYSTEM.md` — токены, типографическая шкала, spacing/radius, layout-паттерны.
- **Реестр Figma-компонентов** (обновляется через `/refresh-figma-snapshot`): `design-system/figma-library-snapshot.json`.
- **Figma file** (mymeet.ai library): `Ma7dtZb6eSHJ5YoaiVEUn2`.
- **GitHub**: https://github.com/avr1l14th/design-lab

Если что-то непонятно — открой `SKILL.md`. Это самый полный документ.

## Принцип короче всего

> Skill не привязан к конкретному формату output. Он применяет mymeet-принципы (токены, типографика, плотность, цветовая дисциплина, ToV, референсы) к **любому** артефакту, который ты попросишь сделать. Технологию для output Claude выбирает сам исходя из задачи.

То есть ты говоришь **что** нужно — Claude разбирается **как** это лучше всего сделать в нашем дизайне.
