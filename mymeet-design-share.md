# Skill «mymeet-design» — генерация чего угодно в дизайне mymeet.ai

> Документ для шеринга любому члену команды.

---

## Что это

`mymeet-design` — это **Claude Code skill** который применяет правила и философию дизайн-системы mymeet.ai к **любому** артефакту, который ты попросишь сделать.

Когда ты пишешь Claude «сделай в нашем дизайне X» или «обнови X под mymeet» — он:

1. Применяет mymeet-философию (типографика Inter Medium 13 как UI body, brand blue только в accent'ах, border-only cards без shadow, filled heroicons, плотность Stripe-style, ToV без «Вы» с большой)
2. Использует токены из дизайн-системы (цвета, spacing scale, radius, font sizes)
3. Выбирает технологию под задачу и генерит результат

Артефакт может быть **любым** — документ, презентация, КП, рассылка, чек-лист, диаграмма, social post, баннер, инфографика, what-have-you. Skill не ограничен списком форматов.

Главное — результат выглядит **родным** для всех остальных артефактов команды.

## Установка (один раз, ~2 минуты)

### 1. Поставь Claude Code

Если ещё нет — поставь Claude Code:
```bash
curl -fsSL https://claude.ai/install.sh | bash
```
Запусти `claude` в терминале, залогинься через браузер.

### 2. Положи skill в нужную папку

Тебе нужен один файл — `SKILL.md`. Положи его сюда:
```
~/.claude/skills/mymeet-design/SKILL.md
```

Создай папку если её нет.

Способ скачать файл — на выбор:

**Способ A (вручную, проще):**
1. Открой [SKILL.md на GitHub](https://github.com/avr1l14th/design-lab/blob/main/.claude/skills/mymeet-design/SKILL.md)
2. Нажми кнопку **Download raw file** (или скопируй содержимое)
3. Сохрани файл в `~/.claude/skills/mymeet-design/SKILL.md`

**Способ B (одной командой в терминале):**
```bash
mkdir -p ~/.claude/skills/mymeet-design && \
curl -fsSL https://raw.githubusercontent.com/avr1l14th/design-lab/main/.claude/skills/mymeet-design/SKILL.md \
  -o ~/.claude/skills/mymeet-design/SKILL.md
```

Эта команда создаёт папку и скачивает файл. Когда skill обновится — повторишь её же, файл перезапишется свежей версией.

### 3. Проверь

Запусти Claude Code где угодно (в любой папке на компе):
```bash
claude
```

Введи `/skills` или начни печатать `/mymeet-` — должна появиться подсказка `/mymeet-design`.

## (Опционально) Если хочешь чтобы Claude работал с Figma напрямую

По умолчанию skill применяет mymeet-стиль к артефактам в HTML, документах, рассылках, диаграммах — **без** Figma. Этого хватает для большинства задач (рассылки, лендинги, КП в виде HTML/PDF, чек-листы, инфографика).

**Подключи Figma** если хочешь чтобы Claude мог:
- Создавать Figma-фреймы прямо в нашем дизайн-файле с инстансами из библиотеки
- Делать слайды-карточки для соцсетей в Figma
- Биндить переменные библиотеки (цвета, spacing) к нодам
- Видеть live состояние библиотеки без скачивания файлов

**Что нужно:**

1. **Поставь Figma Desktop App** — https://www.figma.com/downloads/. Залогинься в свой аккаунт.

2. **Получи доступ к команде mymeet.ai в Figma** — попроси у дизайнера / админа добавить тебя в Figma team mymeet.ai.

3. **Подключи Figma MCP к Claude Code:**
   ```bash
   claude mcp add figma
   ```
   Следуй инструкциям (один раз настроить).

4. **Скачай дополнительный файл** `figma-library-snapshot.json` рядом со SKILL.md:
   ```bash
   curl -fsSL https://raw.githubusercontent.com/avr1l14th/design-lab/main/.claude/skills/mymeet-design/figma-library-snapshot.json \
     -o ~/.claude/skills/mymeet-design/figma-library-snapshot.json
   ```
   В нём — точные ключи всех компонентов библиотеки. Claude использует их для программного импорта компонентов в Figma.

После этого Claude сможет генерить артефакты прямо в Figma — например, `/mymeet-design сделай Instagram-карточку про новую фичу в Figma` создаст фрейм 1080×1080 с инстансами Logo и Badge из библиотеки.

## Как пользоваться

### Через слеш-команду (надёжный путь)

```
/mymeet-design <твой запрос>
```

Примеры:
- `/mymeet-design обнови этот документ под наш дизайн` (приложи PDF/DOCX к чату)
- `/mymeet-design собери презентацию про X в нашем стиле`
- `/mymeet-design свёрстай страницу про фичу транскриптов`
- `/mymeet-design сделай шаблон письма пользователям`
- `/mymeet-design нарисуй диаграмму процесса в нашем стиле`
- `/mymeet-design свёрстай чек-лист в стиле mymeet`
- `/mymeet-design сделай инфографику про наши тарифы`

### Без слеша (auto-invoke по фразе)

Skill активируется автоматически если в запросе есть «в нашем дизайне», «mymeet», «в стиле mymeet», «переверстай под наш дизайн»:

```
Обнови этот отчёт под mymeet
Собери slide-карточку про новую интеграцию в нашем стиле
Сделай в нашем дизайне инструкцию для саппорта
```

Если auto-invoke не сработал — вызови явно через `/mymeet-design`.

## Что Claude сделает

1. **Контекст** — прочитает SKILL.md (все правила и токены inline в нём).
2. **Решение** — определит какая технология лучше подходит под твой запрос (HTML с print-стилями для документа, table-based HTML email, SVG для диаграммы, plain HTML для лендинга, etc.).
3. **Спросит куда сохранить** если неясно — текущая папка / Desktop / новый файл / что-то ещё. Не предполагает структуру проекта.
4. **Генерация** — собирает артефакт с правильной типографикой (Inter), цветами из токенов mymeet, иконками filled 16/20, mymeet ToV.
5. **Output** — путь до файла или preview URL, скриншот, что использовано.

## Где работать

Skill **self-contained** — не привязан ни к какому репо или проекту. Запускай Claude в любой папке:

- На Desktop — для одноразового артефакта
- В своей рабочей папке — добавить что-то в стиле mymeet
- В новой папке под клиента/задачу

Skill сам разберётся куда сохранять файлы (спросит или предложит разумный default).

## Если что-то не так

| Проблема | Что делать |
|---|---|
| `/mymeet-design` не в списке `/skills` | Проверь что файл `~/.claude/skills/mymeet-design/SKILL.md` есть. Перезапусти Claude Code (`/exit` + `claude`). |
| Skill не активируется auto — Claude генерит «по своему» | Вызови явно через `/mymeet-design <запрос>` |
| Что-то не в стиле mymeet (например stroke иконки вместо filled, или sidebar 240 вместо 280) | Попроси Claude поправить с указанием конкретного правила. Skill знает все hard rules. |
| Хочешь обновить skill (вышла новая версия) | Повтори Способ B из установки — команда `curl` перезапишет файл свежим. Или скачай заново вручную по Способу A. |

## Принцип короче всего

> Skill применяет mymeet-принципы (токены, типографика, плотность, цветовая дисциплина, ToV, референсы Linear/Notion/Vercel) к **любому** артефакту, который ты попросишь сделать. Технологию для output Claude выбирает сам исходя из задачи. Работает в любой папке.

То есть ты говоришь **что** нужно — Claude разбирается **как** это лучше всего сделать в стиле mymeet.
