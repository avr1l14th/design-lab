# Подключение к Design Lab

Привет! Это инструкция, как подключиться к нашей дизайн-лаборатории и катнуть свой прототип.

## Что мы делаем

Один общий репо, у каждого автора — свои папки с прототипами. Каждый прототип = папка `app/<slug>/`. Любой push в ветку `main` автоматом обновляет сайт на `https://avr1l14th.github.io/design-lab/`.

Стек: Next.js 16 (App Router, static export) + TypeScript + Tailwind CSS 4 + Framer Motion. Это тот же стек, что у прода — поэтому код анимаций и интеракций потом разрабы копируют один-в-один.

## Раздел 1 — Подготовка (один раз, ~15 минут)

### 1.1. Принять приглашение в репо

Открой письмо от GitHub или зайди на `https://github.com/avr1l14th/design-lab/invitations` → **Accept**.

### 1.2. Установить Homebrew (если ещё нет)

Открой Terminal (Cmd+Space → "Terminal") и выполни:

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

Если уже стоит — `brew --version` покажет версию, пропускаешь шаг.

### 1.3. Установить Node 20 и Git

```bash
brew install node@20 git
brew link node@20 --force
node -v
```

`node -v` должен вывести `v20.x.x`. Это важно: GitHub Action собирает сайт именно на Node 20 — на других версиях локально может работать, а при деплое падать.

### 1.4. Настроить Git один раз

Имя и почта попадут в коммиты — пиши свои:

```bash
git config --global user.name "Имя Фамилия"
git config --global user.email "ты@example.com"
```

### 1.5. Залогиниться в GitHub из терминала

Самый простой способ — через GitHub CLI, без возни с SSH-ключами:

```bash
brew install gh
gh auth login
```

Выбирай по очереди: **GitHub.com** → **HTTPS** → **Login with a web browser** → скопируй код → откроется браузер, вставь код, авторизуйся.

### 1.6. Клонировать репо

```bash
cd ~
git clone https://github.com/avr1l14th/design-lab.git
cd design-lab
npm install
```

Последняя команда поставит зависимости, это займёт минуту-две.

### 1.7. Установить редактор (на выбор)

**Claude Code:**
```bash
npm install -g @anthropic-ai/claude-code
cd ~/design-lab
claude
```

**Cursor:** скачай с `https://cursor.com`, открой в нём папку `~/design-lab`.

### 1.8. Проверить, что всё работает

```bash
npm run dev
```

Открой `http://localhost:3000` — должна появиться главная лаборатории с прототипами. Если открылось — всё ок, готов работать. `Ctrl+C` в терминале, чтобы остановить.

## Раздел 2 — Как добавить свой прототип (каждый раз)

### 2.1. Подтяни свежие изменения

Перед началом работы — всегда:

```bash
cd ~/design-lab
git pull
```

### 2.2. Создай папку под свой прототип

В `app/<my-slug>/page.tsx`. Slug — kebab-case на латинице, уникальный (не повторяй существующие). Минимальный пример:

```tsx
// app/my-feature/page.tsx
export default function MyFeaturePage() {
  return <main className="p-8">My feature</main>;
}
```

### 2.3. Зарегистрируй прототип

В файле `prototypes-registry.ts` добавь объект в массив `prototypes`:

```ts
{
  slug: "my-feature",
  title: "Название",
  description: "Что тут",
  tags: ["motion"],
  updatedAt: "2026-04-27",
},
```

`updatedAt` — дата в формате `YYYY-MM-DD`. Без записи в реестре прототип не появится на главной.

### 2.4. Проверь локально

```bash
npm run dev
```

- Главная (`http://localhost:3000`) — должна появиться карточка твоего прототипа.
- Сам прототип — `http://localhost:3000/my-feature`.

### 2.5. Запушь в main

```bash
git add .
git commit -m "add my-feature prototype"
git push
```

### 2.6. Дождись деплоя

Зайди в репо во вкладку **Actions** — там видно, как идёт билд (1–2 минуты). Когда зелёная галочка — открой `https://avr1l14th.github.io/design-lab/my-feature/` и проверь, что всё на месте.

## Раздел 3 — Если используешь Claude Code или Cursor

**Важно:** в корне репо лежит `AGENTS.md` с предупреждением. У нас Next.js версии 16 — у неё breaking changes по сравнению с тем, что AI знает по своим тренировочным данным. AI обязан читать `node_modules/next/dist/docs/` перед написанием кода.

Если AI начнёт писать «как обычно» — попроси сначала открыть `AGENTS.md` и доку Next 16.

**Удобный первый промпт для нового прототипа:**

> Прочитай `AGENTS.md` и `README.md`, посмотри как устроен `app/search-filters/` как референс, и сделай мне новый прототип `<feature>` по такому-то макету.

## Раздел 4 — Конвенции и правила

### Что трогать можно

- Свою папку `app/<my-slug>/` — что угодно: компоненты, хуки, моки, ассеты внутри.
- `prototypes-registry.ts` — добавить **свою** запись (чужие не трогать).

### Что трогать не надо (без согласования)

- `app/page.tsx` — главная страница лаборатории.
- `app/layout.tsx`, `app/globals.css` — общий каркас.
- `next.config.ts`, `.github/workflows/`, `package.json` — инфраструктура.
- Чужие папки прототипов в `app/`.

### Стек, которым пользуемся

- **Tailwind CSS 4** — для стилей.
- **Framer Motion 12** — для анимаций. Важно: совпадает с продом, поэтому разрабы потом копируют код напрямую.
- Никаких новых тяжёлых зависимостей без согласования.

## Раздел 5 — Если что-то сломалось

- **`git pull` пишет про конфликт** — не пытайся чинить сам, напиши в чат.
- **`npm run dev` ругается на типы или импорт** — обычно AI напутал. Попроси его прочитать `AGENTS.md` и доку Next 16, потом переписать.
- **Action упал в красное** — открой вкладку Actions, посмотри лог. Обычно ошибка типов или сборки. Локально воспроизведи через `npm run build`, почини, запушь ещё раз.
- **Сайт не обновился** — GitHub Pages кэширует, обнови страницу с `Cmd+Shift+R`. Если через 5 минут пусто — проверь, что Action прошёл зелёным.
