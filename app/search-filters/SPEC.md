# Поиск и фильтры встреч — продуктовое описание

## Контекст

Список встреч в mymeet.ai разрастается по мере использования продукта — особенно в командных workspace'ах с несколькими авторами и десятками встреч в неделю. Чтобы пользователь мог быстро вернуться к нужной встрече, на экране встреч появляются **поиск** и **фильтры**, работающие совместно.

Прототип: `https://avr1l14th.github.io/design-lab/search-filters/`

---

## Обзор

Рядом с вкладками встреч добавляются две кнопки: **поиск** и **фильтры**.

- Поиск — раскрывается в инпут поверх top-bar; ищет по названию встречи в рамках workspace.
- Фильтры — открывают popover с тремя категориями: источники, авторы, дата.

Поиск и фильтры **независимы** и применяются совместно (логика AND): результат должен удовлетворять и поисковому запросу, и активным фильтрам.

Фильтрация и поиск работают по **всем встречам** workspace, включая раздел «Доступные мне» (встречи, к которым у пользователя есть доступ, но он не автор).

## Поиск

### Активация

В дефолтном состоянии — иконка поиска 36×36 рядом с вкладками. По клику (или Enter/Space с фокуса) — иконка превращается в полноценный инпут на ширину top-bar.

При открытии поиска:
- Заголовок «Встречи» меняется на «Поиск».
- Quick-tabs «Все встречи / Мои встречи / Доступные мне» уезжают вправо и скрываются — освобождая место под инпут.

При закрытии (Esc или крестик в инпуте):
- Запрос очищается.
- Заголовок и quick-tabs возвращаются на место.

### Логика поиска

- Поиск ведётся по полю **«название встречи»**. Транскрипт, участники, теги — не индексируются.
- Совпадение — **подстрока, без учёта регистра**. Запрос «синк» найдёт «Дизайн синк», «Девсинк», «Еженедельный синк продуктовой команды».
- Лишние пробелы в начале/конце запроса игнорируются.
- Запрос с одними пробелами эквивалентен пустому запросу.

### Состояния инпута

| Состояние | Что показывается |
|---|---|
| Закрыт | Иконка поиска 36×36 |
| Открыт, инпут пустой | Полноширокий инпут с placeholder «Поиск по названию встречи», под ним — empty-state |
| Открыт, есть текст | Слева — иконка поиска, в инпуте — текст, справа — крестик «Очистить» |
| Идёт поиск | Скелетон-группы вместо результатов (~700 ms) |
| Найдено | Список результатов, сгруппированный по дате |
| Не найдено | Empty-state «Не удалось ничего найти» |

### Сохранение состояния

- Текст инпута **очищается** при закрытии поиска (Esc / крестик).
- При переходе на карточку встречи и обратно — поиск остаётся открытым с введённым запросом.
- История запросов не ведётся.

### Hotkeys

- **Esc** — закрыть поиск, очистить запрос, вернуться на дефолтный экран встреч.
- **Enter / Space** на иконке поиска (когда инпут свёрнут) — раскрывает инпут с фокусом.

## Фильтры

### Активация

Кнопка фильтров 36×36 рядом с поиском. В экране поиска кнопка также остаётся справа сбоку.

По клику открывается **popover** с главным меню из трёх пунктов:
- Источники
- Авторы
- Дата

При наведении на пункт — справа выезжает **подменю** с выбором значений. В экране поиска подменю открываются **налево**, чтобы не уходить за край viewport.

### Источники

Чекбоксы со списком ВКС-платформ. Показываются только те источники, **которые подключены пользователем**:

- Загружено
- Google Meet
- Расширение
- Zoom
- Я.Телемост
- МТС Линк
- Microsoft Teams
- Контур.Толк
- Jitsi
- SaluteJazz
- TrueConf

### Авторы

Чекбоксы со списком авторов в workspace. Аватар (инициал на цветном фоне) + email.

### Дата

Календарь с выбором диапазона:
- Один клик → начальная дата (выделяется кружком).
- Второй клик по другой дате → диапазон (между ними подсветка).
- Повторный клик по той же одиночной дате → сброс.
- Будущие даты выбрать нельзя (дисабл).
- Применяется по нажатию кнопки **«Применить»** (кнопка появляется при наличии изменений). Исключение: если до этого был установлен диапазон и пользователь его сбросил — изменение применяется сразу без «Применить».

### Логика комбинирования

- **Между типами** (Источники / Авторы / Дата) — **AND**: встреча должна одновременно удовлетворять всем активным условиям.
- **Внутри одного типа** — **OR**: если выбраны Zoom и Google Meet, показываются встречи из любого из них.
- Сброс по одному фильтру отдельной кнопкой не реализован — нужно вручную снять чекбоксы или очистить дату.
- Кнопка **«Очистить фильтры»** в главном меню сбрасывает всё разом.

### Индикаторы активности

- Иконка фильтров меняется на иконку с **синей точкой**, если активен хотя бы один фильтр.
- В главном меню у активного пункта (например, «Источники» с выбранными значениями) — синяя точка-индикатор справа.
- Числовой счётчик активных фильтров не показывается.

### Поведение при активных фильтрах

- Quick-tabs «Все встречи / Мои встречи / Доступные мне» **скрываются** при наличии любого активного фильтра. Возвращаются после «Очистить фильтры».
- Активные фильтры **сохраняются** при переходе на карточку встречи и обратно — пользователь возвращается к тому же отфильтрованному списку. Фильтры не сбрасываются автоматически; чтобы убрать их, нужно сделать это вручную.

## Поиск + фильтры вместе

Поиск и фильтры независимы и применяются совместно (AND).

| Что задано | Результат |
|---|---|
| Только запрос «синк» | Все встречи с «синк» в названии, любой источник/автор/дата |
| Только фильтр «Источник: Google Meet» | Все встречи в Google Meet |
| Запрос «синк» + фильтр «Источник: Google Meet» | Встречи с «синк» в названии, прошедшие в Google Meet |
| Ничего | Все встречи (или дефолтная вкладка «Все встречи») |

## Результаты

### Строка результата (анатомия)

Каждая встреча отображается строкой высотой **72 px** на белом фоне.

Слева направо:

1. **Превью** 80×48 px со скруглением 4 px. Состояние зависит от типа встречи:
   - Видеокадр + иконка play.
   - Серая подложка с иконкой звука (audio-only).
   - Плашка «ERROR» (ошибка обработки).
   - Плашка «NEW» с blur (новая, не открытая).
   - Иконка legacy.
2. **Текст встречи** (≈446 px ширины):
   - Название (truncate при переполнении), 13 px medium.
   - Время начала · длительность, например `15:00 · 60 min`, 12 px серый.
3. **Автор** (≈180 px): аватар 16×16 с инициалом + email, 12 px (truncate).
4. **Источник** (≈180 px): иконка платформы 14×14 + название, 12 px (truncate).

### Группировка и сортировка

Результаты сгруппированы по **дате**. Заголовок группы: основной лейбл + день недели.

Лейблы:
- Сегодня — «Сегодня» + день недели (например, «Пятница»).
- Вчера — «Вчера» + день недели.
- Раньше — «24 апреля» (число + месяц в родительном падеже) + день недели.

Сортировка:
- Группы — по дате ↓ (свежие сверху).
- Внутри группы — по времени начала ↓ (более поздние сверху).

Пагинации нет — все совпадения показываются единым прокручиваемым списком.

### Loading state

Во время поиска или применения фильтров вместо результатов показываются **скелетон-группы**: 3 группы по 3 строки, со skeleton-заголовком даты.

- При изменении поискового запроса — ~700 ms.
- При изменении фильтров — ~350 ms.

Бары имеют shimmer-анимацию. Длина title-баров рандомизирована для визуальной реалистичности.

### Empty states

| Когда | Иллюстрация | Заголовок | Подсказка |
|---|---|---|---|
| Поиск открыт, инпут пустой | empty-cat (видео) | Поиск по встречам | Введите ключевые слова — найдем все совпадения в этом рабочем пространстве |
| Запрос введён, нет совпадений (фильтры неактивны) | no-results-monkey | Не удалось ничего найти | Попробуйте другой запрос или смените рабочее пространство |
| Активны фильтры (с запросом или без), нет совпадений | no-results-monkey | Не удалось ничего найти | По выбранным фильтрам встречи не найдены, измените или очистите фильтры |

## Анимации (для разработки)

### Раскрытие инпута поиска (открытие)

| Элемент | Свойство | Длительность | Easing |
|---|---|---|---|
| Контейнер инпута | max-width 80px → 100% | 500 ms | cubic-bezier(0.22, 1, 0.36, 1) |
| Заголовок «Встречи» → «Поиск» | opacity (кросс-фейд) | 280 ms | cubic-bezier(0.22, 1, 0.36, 1) |
| Quick-tabs (скрытие) | opacity 1→0 + translateX(16px) + max-width | opacity 180 ms / max-width 500 ms | cubic-bezier(0.22, 1, 0.36, 1) |
| Текст внутри инпута | opacity 0→1 (delay 220 ms) | 240 ms | cubic-bezier(0.22, 1, 0.36, 1) |

### Закрытие поиска (Esc или крестик)

| Элемент | Свойство | Длительность | Easing |
|---|---|---|---|
| Контейнер инпута | max-width 100% → 80px | 500 ms | cubic-bezier(0.22, 1, 0.36, 1) |
| Текст инпута | opacity 1 → 0 | 120 ms | linear |
| Quick-tabs (возврат) | opacity 0→1 (delay 160 ms) + max-width | opacity 320 ms / max-width 500 ms | cubic-bezier(0.22, 1, 0.36, 1) |

### Лоадеры

- При изменении поискового запроса: скелетоны на ~700 ms.
- При изменении фильтров: скелетоны на ~350 ms.

## Out of scope (V2)

Не вошло в MVP, может появиться позже:

- Поиск по транскрипту, участникам, тегам, описанию встречи.
- Глобальный hotkey (Cmd+K / Ctrl+K).
- История последних запросов.
- Сохранённые фильтры / Saved Views.
- Дополнительные типы фильтров: длительность, теги, тип встречи.
- Подсветка совпадений в результатах поиска.
- Сортировка по другим критериям (релевантность, длительность).
- Мобильная адаптация (текущий прототип — desktop-only).
