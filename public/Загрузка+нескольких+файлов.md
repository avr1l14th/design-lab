# Загрузка нескольких файлов — продуктовое описание

## Контекст

В mymeet.ai при импорте существующей записи встречи сейчас можно добавить только один файл за раз. Активные пользователи с пачкой записей за неделю или накопленным архивом вынуждены повторять модалку раз за разом: открыть → выбрать → дождаться → закрыть → открыть. Это ощутимый friction и одно из частых обращений в поддержку.

Чтобы убрать рутину, экран встреч получает обновлённую модалку загрузки, которая принимает **несколько файлов в одну сессию** и запускает обработку всех одной кнопкой.

Прототип: `https://avr1l14th.github.io/design-lab/multi-file-upload/`

---

## Обзор

Кнопка **«Добавить встречу»** в сайдбаре открывает модалку загрузки.

Внутри модалки:
- **DropZone** — зона, в которую можно перетащить файлы или кликнуть для нативного пикера.
- **Очередь файлов** — список добавленных файлов, каждый со своим прогрессом и статусом.
- **Настройки обработки** — селектор AI-отчёта (один на всю пачку) и тоггл авто-распознавания участников.
- **Футер** — счётчик «N файлов (M минут)» и кнопка «Начать обработку».

Юзер набирает очередь, дожидается загрузки всех файлов, нажимает «Начать обработку» — модалка закрывается, в верх группы «Сегодня» в списке встреч вставляются N строк со статусом «Обрабатывается». По мере готовности транскрибации каждая строка превращается в обычную карточку встречи с длительностью.

---

## Открытие и закрытие модалки

Кнопка «Добавить встречу» — 36×36, синяя (#0138C7), в левом сайдбаре. По клику открывается модалка фиксированной ширины 500px, центрированная на экране, с затемнённым backdrop'ом вокруг.

Закрытие модалки:
- Клик по крестику в хедере.
- Клик по backdrop'у вокруг модалки.
- Клавиша **Esc**.

При закрытии модалки во время активных загрузок все in-flight загрузки отменяются, очередь сбрасывается. Confirm-диалог («у вас есть незавершённые загрузки») в этой версии не делаем.

При закрытии focus возвращается на кнопку «Добавить встречу».

## Добавление файлов

### Через DropZone

В дефолтном состоянии DropZone показывает иконку закрытой папки и копи **«Выберите файл или перетащите его сюда»**. Подсказка снизу: **«Любые аудио или видео меньше 3 ГБ»**.

При перетаскивании файла **над любой частью модалки** (не только над самой DropZone) — DropZone подсвечивается синим, иконка папки морфит в открытую (drop-anywhere паттерн как в Linear / Notion / Figma). Drop в любой точке модалки = файлы добавляются.

Клик по DropZone (или Enter / Space с фокуса) — открывает нативный файл-пикер с фильтром `audio/*, video/*` и поддержкой выбора нескольких файлов.

### Поддерживаемые форматы

- Аудио: mp3, wav, m4a, ogg, flac, aac.
- Видео: mp4, mov, m4v, webm, avi, mkv.
- Размер ≤ 3 ГБ на файл.

### Невалидные файлы

Файлы, не прошедшие валидацию, в очередь не добавляются. В правом нижнем углу появляется toast с конкретной причиной:

| Причина | Текст toast'а |
|---|---|
| Неверный формат | N файлов пропущено: неверный формат |
| Превышен размер | 1 файл слишком большой (>3 ГБ) |

Если в одном дропе оба типа отказов — два toast'а stacked. Auto-dismiss через 4000 ms или по клику на крестик.

## Очередь файлов

Появляется под DropZone, когда есть хотя бы один добавленный файл. До 3 строк видны без скролла, дальше — тонкий кастомный скроллбар (7px ширины, серый thumb).

Каждая строка: превьюшка 80×48, имя файла, подпись с длительностью и статусом, кнопки действий справа.

### Состояния файла

| Состояние | Превьюшка | Имя файла | Подпись | Кнопки |
|---|---|---|---|---|
| **Загружается** | серая коробка с прогресс-баром (заливка слева на N%) и текстом «N%» по центру | серый | «X мин · Загрузка...» (серый) | × |
| **Загружено** | картинка для аудио / иконка плёнки для видео | чёрный | «X мин · ✓ Загружено» (зелёный) | × |
| **Ошибка** | как при загрузке, прогресс заморожен на проценте падения | серый | красный текст ошибки | ↻ + × |

### Действия в строке

- **× (удалить)** — строка плавно сворачивается за 220 ms, потом удаляется из стейта. Если шла загрузка — отменяется.
- **↻ (ретрай)** — появляется только в состоянии «ошибка». Сбрасывает status в «загружается», обнуляет прогресс, перезапускает загрузку.

### Auto-scroll

При добавлении новых файлов в очередь, длина которой превышает видимую область, очередь плавно скроллится до низа за 220 ms — чтобы свежедобавленный файл был виден.

### Параллельность загрузок

Все файлы в очереди грузятся параллельно. Можно добавлять новые файлы в момент, когда другие ещё грузятся — они присоединяются к параллельной загрузке.

### Переход «загружается» → «загружено»

Минуты и точка-разделитель в подписи остаются на месте без изменений (только мгновенный flip цвета). Правая часть подписи — лейбл «Загрузка...» — морфит в «✓ Загружено» через blur cross-fade на месте: уходящий лейбл уходит c blur'ом за 180 ms, входящий приходит с задержкой 120 ms за 260 ms.

## Настройки обработки

Под очередью файлов:

- **Селектор AI-отчёта** — один dropdown для всей пачки. По умолчанию «Обычная встреча». Все файлы в текущей сессии получают одинаковый шаблон отчёта.
- **Тоггл авто-распознавания участников** — стандартный switch. Применяется ко всем файлам в пачке.

Per-file настройки в этой версии не делаем.

## Submit — кнопка «Начать обработку»

В правой части футера. Текст: «Начать обработку».

### Состояния

| Состояние | Условие | Стиль |
|---|---|---|
| Disabled | очередь пустая ИЛИ есть незавершённый файл (загружается / ошибка) | бледно-синий (#809BE3), курсор not-allowed |
| Enabled | все файлы в статусе «загружено» И их ≥ 1 | насыщенный синий (#0138C7), курсор pointer |

Переход disabled → enabled — плавный fade фона за 180 ms, чтобы кнопка не моргала между состояниями.

Кнопка обрабатывает только первый клик, дальше игнорирует повторные нажатия до закрытия модалки. На время отправки кнопка визуально нажата (scale 0.97 + opacity 0.85).

### Что происходит на клик

1. Все «загруженные» файлы конвертируются в записи встреч в статусе «обрабатывается».
2. Эти записи вставляются в верх группы «Сегодня» в списке встреч.
3. Запускается транскрибация и пайплайн обработки на бэке.
4. Очередь очищается, модалка закрывается.

## Счётчик в футере

Слева в футере: «N файлов (M минут)».

- Показывается **только когда ≥ 2 файла** в статусе «загружено». При одном файле счётчик скрыт — информация уже видна в самой строке.
- Считаются **только** файлы в статусе «загружено». Загружающиеся и ошибочные не учитываются.
- Русские плюрализационные правила: 1 файл / 2–4 файла / 5+ файлов; 1 минута / 2–4 минуты / 5+ минут (с исключением для 11–14).

---

## Состояние «Обрабатывается» в списке встреч

После сабмита в верх группы «Сегодня» вставляются N строк — по одной на файл. Каждая строка появляется со slide-in анимацией.

В строке:
- Превьюшка как у обычной встречи (audio / video).
- Заголовок — имя файла без расширения.
- Вместо длительности — синяя пульсирующая точка + текст **«Обрабатывается»** (синий).
- Автор: текущий пользователь, источник: «Загружено».

По завершении транскрибации subline плавно меняется на «X мин» через тот же blur cross-fade, что и в очереди файлов.

## Поведение и edge cases

| Кейс | Поведение |
|---|---|
| Drop пустой папки | Ничего не происходит. |
| Удаление файла во время его загрузки | In-flight отменяется, ряд анимированно сворачивается. |
| Закрытие модалки во время загрузок | Все in-flight отменяются, очередь сбрасывается без подтверждения. |
| Все файлы в статусе «ошибка» | Submit неактивен. Юзер должен либо удалить, либо ретраить. |
| Длинное имя файла | Truncate с ellipsis, полное имя — в tooltip. |
| Двойной клик по «Начать обработку» | Срабатывает один раз. |
| Перезагрузка страницы (F5) во время загрузок | Очередь теряется. Persist в localStorage в этой версии не делаем. |

## Доступность

- **Focus trap** внутри модалки. Tab / Shift+Tab не уходят за её пределы.
- **Esc** — закрывает модалку, focus возвращается на кнопку-открыватель.
- **DropZone** активируется Enter / Space (`role="button"`, `tabIndex=0`).
- Все интерактивные иконки (×, ↻, крестик хедера) имеют `aria-label`.
- Toggle авто-распознавания: `role="switch"`, `aria-checked={boolean}`.
- `prefers-reduced-motion: reduce` отключает все transform / opacity-анимации (folder morph, blur cross-fade, slide-in row, exit модалки) — оставляет мгновенные state changes.

| Элемент | Свойство | Длительность | Easing |

| Элемент | Свойство | Длительность | Easing |
|---|---|---|---|

| Элемент | Свойство | Длительность | Easing |

### Submit — переход disabled → enabled

| Элемент | Свойство | Длительность | Easing |
|---|---|---|---|
| Кнопка | background-color #809BE3 → #0138C7 | 180 ms | ease |

### Toast (rejected files)

| Элемент | Свойство | Длительность | Easing |
|---|---|---|---|
| Toast | opacity 0 → 1, translateX 8px → 0 | 200 ms | ease-out |

### Появление строки «Обрабатывается» в списке встреч

|---|---|---|---|
| Строка | opacity 0 → 1, translateY -8px → 0 | 280 ms | cubic-bezier(0.22, 1, 0.36, 1) |

| Элемент | Свойство | Длительность | Easing |
|---|---|---|---|
| Точка | opacity 0.4 ↔ 1 | 1400 ms | ease-in-out, infinite |

|---|---|---|---|
